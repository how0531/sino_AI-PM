# AIPM 功能擴充規劃書 v2(便利性升級)

**日期:** 2026-07-14　**方法:** 四視角平行發想(PM 日常/成員接收端/主管可見性/自動化)+平台能力研究(Microsoft Learn 官方查證),彙整去重後按優先級排序。

## ⚡ 兩個平台級發現(先讀,改變原規劃)

### 發現 1:Copilot Studio 原生事件觸發已 GA——排程不必再靠 Power Automate

官方觸發庫內建 **Recurrence(排程)**、**When a task is completed(Planner)**、**When an email arrives** 等,agent 直接加觸發器即可主動執行(前提 generative orchestration,AIPM 已符合)。原本 A6「排程 Flow 藍圖(Power Automate 人工建)」可改用原生觸發,**不用等服務帳號**。
注意:①觸發以 maker(郭文浩)憑證執行;②每次觸發吃 Copilot Credits(一次約 15-40);③發布後觸發自動生效,上線前必測。
出處:learn.microsoft.com/microsoft-copilot-studio/authoring-triggers-about

### 發現 2:負責人 GUID 反查有解——已知最大體驗硬傷可修

Planner 任務負責人的 GUID 就是 **AAD 使用者 objectId**,O365 使用者連接器「**取得使用者設定檔 (V2)**」可直接用 GUID 反查 displayName/email(另 Work IQ MCP 的 getUserProfile 也支援,但 preview)。修好後:查進度顯示人名、催辦自動帶 email、KPI 按人名分負載——之前記錄的「催辦需 PM 手動給收件人」限制可解。**需先用團隊 3 人實測 GUID 反查成功再全面套用。**

### 觀察名單(Preview,先在測試 agent 驗,不上生產)

- **Work IQ MCP**(User/Copilot 節點,租戶已可見):getUserProfile 反查 GUID、全 M365 內容檢索。Preview+需 M365 Copilot 授權。
- **Code interpreter**:KPI 改用 Python 沙箱做「確定性計算」取代 LLM 心算;分析電子市佔 Excel。Preview+credits。
- **Deep reasoning**:僅美/歐 region+資料落地無承諾——**大概率開不了且合規紅線,先不碰**。
- **Computer use**(GA):打通只有 UI 沒 API 的內網系統。要專用機器+資安審查,後期選項。
- **M365 Copilot channel**:同一 agent 出現在 Word/Excel/Outlook 的 Copilot 介面。GA 但要 admin 核准+授權,擴編時走。

- **第 1 週**:P0 全部(6 項,合計 ~2 天)→ 立即有感,GUID 修復解鎖下游

## 風險備忘

1. 原生觸發用 **maker 憑證**執行,分享 agent 等於分享 PM 的資料可見面——擴編前要重新評估。
2. Credits:全開自動化月估 USD15-40,但**必須先設 PPAC 上限**(125% 超額 agent 直接停用)。
3. Preview 功能(Work IQ/code interpreter)官方明文不建議生產;AIPM 已上線,一律先開測試 agent 驗證。
4. 所有新外送場景維持「一次確認」原則;唯排程推播(唯讀彙總、只發 PM 本人)可豁免。
